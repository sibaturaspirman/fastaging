Improvement history list

Improvement history of firmware Ver. 1.005 (October 2024)

No. Contents
1 If a partner encoder was used, a 32-bit restore solved the problem.
2 When the electronic gear of the MR-J5 (W)-G is set to a value other than the following, "1DC8H: Servo parameter invalid" does not occur and the value may not be automatically rewritten.
- Electronic gear numerator: 1,Electronic gear denominator: 1
- Electronic gear numerator: 16,Electronic gear denominator: 1

Improvement history of firmware Ver. 1.004 (May 2024)

No. Contents
1 When there is a significant difference between the commanded pulses and feedback pulses on the degree axis, the issue of the [Md.101] actual current value becoming invalid has been resolved.

Improvement history of firmware Ver. 1.003 (October 2023)

No. Contents
1 Resolves an issue that caused a mismatch with the CPU unit time when data linking.
2 Resolves an issue that registered speed limit value over (0D51H) twice in the event history when the error occurred during continuous positioning control/ continuous trajectory control execution on command generation axis and restart was executed during the previous positioning data execution.
3 Resolves an issue that caused hardware error (3C00H) when cam data was written by the cam data operation function after parameter initialization.
4 Resolved an issue that could cause Advanced Synchronization Control to stop "in Analysis" when started on multiple axes.
5 Resolved an issue that might cause a servo error (AL. 35.1: Abnormality of command frequency) when the current value of the input axis was changed between the start of advanced synchronization control and the start of synchronization control.

Improvement history of firmware Ver. 1.002 (April 2022)

No. Contents
1 The setting range of "[Cd.13] Positioning operation speed override" is changed from 1 to 300% to 0 to 300%.
2 The setting range of "[Pr.2] to [Pr.4] Electronic gear" is extended.
3 By using the CC-Link IE TSN-compatible MR-J5-G servo amplifier with firmware version D0 or later, fixed a problem in which the absolute position restored when the power was cycled after an encoder initial communication error occurred in the servo amplifier would deviate.

Improvement history of firmware Ver. 1.001 (October 2021)

No. Contents
1 Fixed a problem in which the data points being executed would not be restored when a step was continued if continuous positioning operation was executed from block start in step mode.
2 Fixed a problem in which the accumulated clutch slip value would be incorrect when the clutch smoothing method was operated as a slip amount method (linear: input amount tracking).
3 Fixed a problem in which a servo alarm (AL. 09E. 8: Master station error detection alarm) would be detected when synchronous control was started while [Pr. 422] Auxiliary shaft clutch control setting was set outside the setting range (0056H).
4 Fixed a problem in which [Md. 101] Actual current value would bounce during speed control at a high speed of approximately 6000 rpm in fixed position stop speed control.
5 Fixed a problem in which Input axis speed display over (0E42H) would occur when home position return control was executed on the virtual servo amplifier axis.
6 Fixed a problem in which the timing of the completion pop-up of [Md.191] Flash ROM write, parameter initialization request acceptance information, [Md.192] Flash ROM write, parameter initialization progress information, and the engineering tool would deviate.
7 Fixed a problem in which the date and time of current value history power-off data would not be displayed correctly when the power was cycled after detection of an incorrect home position data.
8 Separated some warning codes and error codes.
