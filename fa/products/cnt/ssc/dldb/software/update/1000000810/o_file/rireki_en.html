Improvement history list (MELSEC iQ-R Motion module FB library)

Revision history of PLCopen Motion Control FB Ver. 10K (November 2024)

No. Contents
1 Support for mcRampIn in the start mode of MC_CamIn (Cam Operation Start).

Revision history of PLCopen Motion Control FB Ver. 09J (April 2024)

No. Contents
1 Resolved a problem in which MCv_AdvCamSetPositionCalc (Advanced Synchronous Control Cam Set Position Calculation) caused a conversion error in GX Works3 Ver.1.105K or later.
2 Resolved a problem in which motion control FB did not start when sequencer ready was switched off during motion control FB start-up, even if sequencer ready was switched on again.
3 Problem solved where an error code was not output when Execute/Enable was turned on during Ready OFF.

Revision history of PLCopen Motion Control FB Ver. 07H (February 2023)

No. Contents
1 Supports MCv_MovePositioningData (Multiple Axes Positioning Data Operation).

Revision history of PLCopen Motion Control FB Ver. 06G (December 2022)

No. Contents
1 Supports MCv_AdvancedSync (Advanced Synchronous Control).
2 Supports MCv_AdvPositionPerCycleCalc (advanced Synchronous control 1-cycle current position calculation).
3 Supports MCv_AdvCamSetPositionCalc (advanced synchronous control cam command current position calculation).
4 Added ENUM (global label) for use with motion control FB.
5 Resolves the problem that if Enable type motion control FB is written during RUN under the condition that Enable/JogForward/JogBackward becomes TRUE, the motion control FB does not start until Enable/JogForward/JogBackward is changed from FALSE to TRUE again.

Revision history of PLCopen Motion Control FB Ver. 05F (September 2022)

No. Contents
1 Resolves an issue where motion control FB may not work if X1 is turned on due to factors other than Motion Module's sync flag.

Revision history of PLCopen Motion Control FB Ver. 04E (June 2021)

No. Contents
1 Resolve issues that FB's Input Variable and output variable's Reflection timing do not partially comply with PLCopen standards.
2 Solves a problem that causes illegal operation when I/O No. of Motion module that results in Target specified in I/O Variable of motion control FB is changed halfway.

Revision history of PLCopen Motion Control FB Ver. 03D (September 2020)

No. Contents
1 Performance improvements.
2 Resolves issues with error (H0400) when MCFB are written during RUN-in-progress writes during MCFB.
3 Solves the issue where MCFB may not operate if Busy (execution command) (Execute) is turned OFF after Execute is ON.
4 Resolves an issue where or less runs a MCFB and Busy is under control (Active) and Active is output as running (Busy).
- MC_GearIn
- MC_CombineAxis
- MCv_SmoothingFilter
- MCv_BacklashCompensationFilter
5 Resolve issues when running MC_TouchProbe and running (Busy) might chattering after leading edge detection of execution command (Execute).
