Improvement history of Motion Control Software SWM-G Ver. 1.102C (March 2025)

No. Contents
1 Fixed a problem in which a profile exceeding the maximum jerk was generated when the target position was close to the current position and the command speed had not been reached in the jerk restriction motion profile.
2 Fixed a problem in which the Write buffer indicator would not update on the SWMOS file log screen.
3 Fixed a problem in which the maximum number of compensation points could be exceeded in 2D pitch error compensation.
4 Fixed a problem in which an error would not occur, and the trigger conditions would not change when an absolute value was selected with -2147483648 entered for the condition value in the MotionScope trigger settings.
5 Changed the amount of local memory allocation on the installer RTX setting screen.
6 Fixed a problem in which data named Cal_ could not be read properly after being logged, saved to a CSV file, and then read again when the communication cycle was less than 1 ms in MotionScope.
7 Fixed a problem in which if only the DIO chart AxisFlag was set, axis data that was not the set axis number would also be received in the MotionScope setting tab.
8 Fixed a problem in which values outside the setting range could be set in the AIO chart setting in MotionScope.
9 Fixed a problem in which the layout display would reset when transitioning from the layout screen to the setting screen in MotionScope.
10 Fixed a problem in which an error message would not be displayed when switching a hidden chart in the MotionScope chart display switching.
11 Fixed a problem in which a non-existing chart number could be specified, or an existing chart number could be specified in MotionScope chart display switching.
12 Expanded the length of the specified path when saving settings to a maximum of 260 characters in MotionScope.
13 Improved so that the following parameters can be logged in SWMOS.

14 Fixed a problem in which the correct axis number was not read in 2D chart settings in MotionScope.
15 Fixed a problem in which deceleration at stopping would not be the specified value set in the position control Jog operation and speed control. This occurs with the following speed profile types: Scurve, JerkRatio, AdvancedS, JerkRatioFixedVelocityT, JerkRatioFixedVelocityS.
16 Fixed a problem in which the Jerk limit text box on the single-axis control screen would sometimes become hidden under specific conditions such as engine restart or communication start/stop in SWMOS.
17 Improved so that when axes are not assigned when communications starts, the communication will fail to start in each communication platform.

Improvement history of Motion Control Software SWM-G Ver. 1.101B (December 2024)

No. Contents
1 Fixed a problem in which the homing parameter Clear Home Done On ServoOff would not be enabled on a certain screen in SWMOS.
2 Fixed a problem in which reciprocating operation would sometimes unintentionally resume after a communication reset and servo-on again when the reciprocating operation is interrupted by something other than the stop button for the reciprocating operation (such as communication stop, servo-off) on the SWMOS gantry operation screen.
3 Improved the reciprocating operation button so that it changes color when the reciprocating operation is active in SWMOS.

Improvement history of Motion Control Software SWM-G Ver. 1.100A (September 2024)

No. Contents
1 Deleted SWM-G User Manual from the installer and SWMOS.
SWM-G User Manual is available on this page.
2 Fixed a problem in which the synchronization release timing would be delayed when the master axis was servo-off in the synchronization function.
3 Corrected the description regarding the synchronization function of CoreMotionApi in SWM-G User Manual.
4 Fixed a problem in which homing would not be performed correctly for the following home types when the HomeDirection parameter was changed during homing.
· HSReverseZPulse
· HSOffReverseZPulse
· LSReverseZPulse
· NearLSReverseZPulse
· ExternalLSReverseZPulse
· MechanicalEndDetectionReverseZPulse
5 Fixed a problem in which a function would sometimes return without blocking when the API Wait Until Motion Start parameter was enabled.
6 Fixed a problem in which the rotating axis would sometimes move to the wrong angle after moving to a specified point or position in rotation path interpolation.
7 Fixed a problem in which the command position and the feedback position of the target axis would not be properly initialized during homing in pitch error compensation and 2D pitch error compensation.
8 Modified the following items on the gantry control screen of SWMOS.
(1) Fixed a problem in which each option would be enabled even when the homing option checkbox on the gantry control setting screen was unchecked.
(2) Deleted the step operation button (at the top of the gantry operation screen) and the control for Y-axis and Z-axis, as they were used infrequently.
(3) Fixed a problem in which the JOG operation button (at the bottom of the gantry operation screen) on the gantry operation screen would not check the synchronization status, allowing the single axis to operate without the following axis.
9 Fixed a problem in which version 4 of the CSP+ file could not be read.
10 Updated the CSPP file.
11 Changed specifications so that digital I/O objects would not be automatically mapped to the I/O address space even if they are mapped in PDO.
12 Fixed a problem in which unintended data would sometimes be mapped when input/output size and allocation addresses were defined by directly editing the network setting file.

No. Contents

8 Fixed a problem in which profiles could not be specified correctly for speed control methods [Velocity] and [TimedVel] on the test screen of SWMOS speed control.
9 Fixed a problem in which the message would not display in a SWMOS message window.
10 Fixed a problem in which cells other than "New Axis" were editable in the axis mapping function of DEF file generation in IETSN Configurator.
11 Fixed a problem in which acceleration could not be calculated correctly (0.5 to 2 times the specified value for acceleration depending on the profile type and jerk ratio) when using JerkLimitedSCurve or AdvancedS profile in a FastBlending or Smoothing type interpolation override command.
12 Fixed a problem in which an out of range error occurred in an array index when AddPathIntplWithRotationCommand function was called after over 513 points were specified for the array size of the Point member in PathIntplWithRotationCommand class in .NET library.

· StartPVT

16 Fixed the following problem: when acceleration time profiles (profiles that includeTimeAcc- in their names such as TimeAccTrapezoidal) were overridden by non-acceleration time profiles (such as Trapezoidal), the acceleration parameters specified in the non-acceleration profile were processed as the acceleration time.
17 Fixed the following problem: when a two velocity profile (profiles that include TwoVelocity- in their names such as TwoVelocityTrapezoidal) was overridden by an override function such as OverridePos and OverrideVel, the corresponding non two velocity profile (Trapezoidal, etc.) would be generated.

1. The motionComplete would not be set to 1 after the following interpolation operations were resumed by the Resume function after being suspended by the Pause function near the completion of the operation in order to stop it at the target position: linear interpolation, circular interpolation, helical interpolation, path interpolation, 3D path interpolation, and rotation path interpolation.
2. The motionComplete would not be set to 1 for linear interpolation override and circular interpolation override depending on the completion timing of the original interpolation and override interpolation.
19 Fixed a problem in which the home position would change when servo OFF is performed while backlash compensation, pitch error compensation, or 2D pitch error compensation was enabled.

· StartTrqToVel

1. The tolerance option is set to 0.
2. The enableTolerance is enabled

26 Fixed a problem in pitch error compensation or 2D pitch error compensation where the command position of the compensation axis would suddenly change for the compensation amount only, when re-enabling the compensation after disabling it and before the next communication cycle.
27 Fixed a problem in 2D pitch error compensation where the initial command position calculated at the start of 2D pitch error compensation was not optimum when the Prevent Catch Up Motion parameter was enabled.
28 Fixed the following problem in 2D pitch error compensation where the encoder command position would change one cycle after enabling compensation in cases where compensation is applied to two axes with two channels.
