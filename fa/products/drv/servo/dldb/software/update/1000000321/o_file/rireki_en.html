Improvement history list

Improvement history of firmware version E9 (December, 2024)

There are no major improvements.

Improvement history of firmware version E8 (December, 2024)

No. Description
1 Improved the detection accuracy by changing the detection threshold of Output open-phase error (AL. 139.2) for servo amplifiers with a capacity of 2 kW or more.

Improvement history of firmware version E4 (April, 2024)

No. Description
1 Made improvements so that regeneration operation is performed during the motor less operation.
2 Changed the detection threshold of Overload warning 2 (AL. 0EC) to enhance the detection accuracy.
3 Fixed a problem in which incorrect values may have been saved in the alarm history after the initialization by MR Mode Change.

Improvement history of firmware version E0 (January, 2024)

No. Description
1 Made improvements so that the overshoot amount is minimized when [Pr. PB02.0_Vibration suppression control 1 - Tuning mode selection] is enabled in the speed control mode.

Improvement history of firmware version D8 (June, 2023)

No. Description
1 Fixed a problem in which the position within one-revolution may have been incorrect when multipoint Z-phase scale was used.
2 Improved the estimation accuracy in the high-frequency band of the machine analyzer function.

Improvement history of firmware version D4 (November, 2022)

No. Description
1 Fixed a problem in which Parameter error (AL. 037) would not occur when [Pr. PA02.4] was set to "1" (simple converter is used) for a 400 V class servo amplifier.
2 Fixed a problem in which the name and waveform would not be displayed when a specific measurement data was selected when the drive recorder data was read with the FTP server function.
3 Fixed a problem in which [Pr. PB17 Shaft resonance suppression filter] may have been disabled after the one-touch tuning was performed.
4 Improved the operation when switching the semi closed loop control/fully closed loop control.

Improvement history of firmware version D1 (September, 2022)

There are no major improvements.

Improvement history of firmware version D0 (May, 2022)

No. Description
1 Fixed a problem in which Parameter error (AL. 037) would not occur when [Pr. PC44.3 Load-side encoder cable communication method selection] was set to "1" (Four-wire type) for MR-J5-_A.
2 Fixed a problem in which Servo control error (AL. 042) may have occurred when the fully closed loop system was used in the direct drive motor control mode.
3 Fixed a problem in which the value of [Pr. PF67 Backlash nominal value] would not be reflected to the travel distance setting in the backlash estimation function when the power supply was cycled or software reset was performed.

Improvement history of firmware version C4 (December, 2021)

No. Description
1 Improved the temperature judgment value for detecting Main circuit device overheat alarm (AL. 045) and Servo amplifier overheat warning (AL. 091) to higher value than before.
2 Fixed a problem in which the upper and lower thresholds of the friction estimation value may not have been shown correctly in the machine diagnosis window of MR Configurator2.
3 Improved the monitor data refresh cycle of the load to motor inertia ratio, peak load ratio, converter effective load ratio, and converter regenerative load ratio.
4 Fixed a problem in which an incorrect firmware version may have been stored when the recorded data of the drive recorder is read by the FTP server function.

Improvement history of firmware version C2 (July, 2021)

There are no major improvements.

Improvement history of firmware version C0 (June, 2021)

No. Description
1 Fixed a problem in which the data before the trigger would not be recorded in the waveform data of the drive recorder when sampling was executed for a long time using the graph function.
2 Fixed a problem in which the communication speed of the initial value of [Pr.NPA12_Communication speed] would not be automatically changed depending on the connected device.

Improvement history of firmware version B8 (January, 2021)

No. Description
1 Improved the accuracy of gear failure diagnosis.

Improvement history of firmware version B6 (January, 2020)

No. Description
1 Fixed a problem in which a part of the graph signal would not be output correctly when the drive recorder data was read with the FTP server function.
2 Fixed a problem in which the negative value of the analog graph signal would not be displayed correctly on GX LogViewer when the drive recorder data was read with the FTP server function.
3 Fixed a problem in which an analog waveform different from the analog waveform described in the manual would be saved when Excessive error alarm (AL. 052) was set in [Pr. PA23_Drive recorder desired alarm trigger setting].
4 Fixed a problem in which changes would not be enabled until the power supply is cycled or software reset is executed when a pushbutton was used to change a parameter without a "*" or "**" before the symbol of the parameter.
5 Fixed a problem in which Memory writing frequency warning (AL. 1F8.1) would occur when Encoder normal communication error 2 (AL. 021) was detected.

Improvement history of firmware version B2 (July, 2020)

No. Description
1 Fixed a problem in which the connection may have been disabled when a FW was updated again without cycling the power supply.
2 Fixed a problem in which the overshoot suppression setting value may not have been reflected as the specification when a direct drive motor was connected.
3 Fixed a problem in which the ABSV may not have been turned off when homing was executed without cycling the power supply after the servo motor was replaced when a batteryless absolute position encoder-mounted servo motor was used.

Improvement history of firmware version B0 (July, 2020)

No. Description
1 Fixed a problem in which USB connection would rarely be lost when the drive recorder was read while MR Configurator2 was connected via USB.
2 Fixed a problem in which Absolute position counter warning (AL. 0E3.2) may have been detected when the power supply was turned on while the servo motor was rotating.

Improvement history of firmware version A7 (January, 2020)

No. Description
1 Fixed a problem in which Homing incomplete warning (AL. 90.1) would rarely be detected at startup when the absolute position detection system was enabled when the encoder multi-revolution counter was -255 or lower.
2 Fixed a problem in which overheat malfunction due to overload of the servo motor may not have been protected when the rotary servo motor HK-KT series were used in a specific combination.
3 Enhanced overheat protection of the rotary servo motor HK-KT/HK-ST series.

Improvement history of firmware version A6 (December, 2019)

No. Description
1 Fixed a problem in which Thermal overload error 4 during operation (AL. 050.3) may have been detected when current exceeding the rated current flowed when a specific linear servo motor (LM-FP2B-06M-1SS0) or direct drive motor (TM-RFM120J10) was connected.

Improvement history of firmware version A5 (October, 2019)

No. Description
1 Fixed a problem in which Servo control error (AL. 042) would occur when the parameter magnetic pole detection was disabled while an absolute position type linear encoder or a direct drive motor with a battery was connected.
2 Fixed a problem in which Instantaneous power failure tough drive warning (AL. 0F0.1) would remain in the alarm history with normal power-off when instantaneous power failure tough drive was enabled.
3 Fixed a problem in which the position control gain would switch when the gain was switched when a parameter of [Pr. PB30 Gain switching - Position control gain] < 1.0 [rad/s] was set.
